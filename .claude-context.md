## Dockerfile Strategy (from nexthis/medusa-coolify)

1. **Build stage**: Install deps, build app → outputs to `.medusa/server/`
2. **Production stage**:
   - Copy `.medusa/` from builder
   - Change WORKDIR to `.medusa/server/`
   - Run `yarn install --production` there
   - Start with `yarn run start`

## Two Services Architecture (Optional)

For production, you can run two services:

1. **medusa-server**: `WORKER_MODE=server`, `DISABLE_ADMIN=false`
2. **medusa-worker**: `WORKER_MODE=worker`, `DISABLE_ADMIN=true`

## Cache Revalidation Setup

When products are updated in admin, the storefront cache needs to be invalidated.

### Backend Subscriber
`src/subscribers/product-updated.ts` - Listens for product updates and calls storefront API

### Frontend API Route
`src/app/api/revalidate/route.ts` - Receives revalidation requests and clears Next.js cache

## Email Notification System

Full documentation: [`docs/EMAIL-SYSTEM.md`](docs/EMAIL-SYSTEM.md)

- **SMTP Provider**: `src/modules/smtp-notification/` — renders branded HTML templates, sends via nodemailer
- **Email Audit**: `src/modules/email-audit/` — logs all emails (queued/sent/failed), stores SMTP settings in DB
- **Templates**: `src/modules/smtp-notification/templates/` — Vita Integral branded (green #5B8C3E, orange #DA763E)
- **Shared helpers**: `shared.ts` — emailWrapper, ctaButton, paragraph, sectionTitle, infoBox, productThumbnail, formatCOP
- **Logo**: PNG format, URL from `EMAIL_LOGO_URL` env var (upload to MinIO, not served from frontend)
- **SMTP config**: Admin UI (Settings > Emails) or env vars (SMTP_HOST, SMTP_PORT, etc.)
- **notifyWithAudit()**: Central helper used by all subscribers — sends notification + audit logs

## Wompi Payment Integration

Full documentation: [`docs/WOMPI.md`](docs/WOMPI.md)

- **Module**: `src/modules/wompi/` — payment link generation, record tracking
- **Webhook**: `src/api/hooks/wompi/events/route.ts` — processes payment results, emails admin + customer
- **Admin UI**: Dashboard at `/admin/wompi`, order sidebar widget
- **Checkout URL**: `https://checkout.wompi.co/l/{payment_link_id}` (constructed, not from API)
- **Env vars**: `WOMPI_PRIVATE_KEY`, `WOMPI_EVENTS_SECRET`, `WOMPI_ENVIRONMENT`, `WOMPI_PAYMENT_MANAGER_EMAIL`
